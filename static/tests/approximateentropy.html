NIST SP 800-22 Statistical Test

Approximate Entropy Test

Quick Summary: The Approximate Entropy Test compares the frequency of overlapping blocks of two consecutive lengths (m and m+1) to measure the unpredictability of the sequence. High approximate entropy indicates high randomness and unpredictability.

What It Tests For

This test measures the irregularity or unpredictability in the sequence by comparing pattern frequencies at different scales:

  • Predictability: Can next bits be predicted from previous patterns?
  • Pattern correlation: Are m-bit and (m+1)-bit pattern frequencies related?
  • Entropy deficiency: Lack of randomness that creates predictable structure
  • Local randomness: Whether short subsequences are sufficiently random

How It Works

Step 1: Choose Block Length m

Select m (typically 2-5), which determines the pattern length to examine:

Typical: m = 2 or m = 3 Minimum sequence length: ~100 bits for m=2

Step 2: Count Overlapping Patterns

Count frequencies of all overlapping m-bit and (m+1)-bit patterns:

For m=2, count: 00, 01, 10, 11 For m+1=3, count: 000, 001, 010, 011, 100, 101, 110, 111

Step 3: Calculate Approximate Entropy

Compute entropy-like measure for both block sizes:

φ(m) = Sum(Ci × log(Ci)) for all patterns Where Ci = frequency of pattern i / total patterns ApEn(m) = φ(m) - φ(m+1)

Step 4: Normalize and Get P-Value

The test statistic is converted to a p-value using chi-square distribution.

Understanding P-Values:
  • p >= 0.01: PASS - Entropy is appropriately high (unpredictable)
  • p < 0.01: FAIL - Too predictable or too irregular

Interpreting Results

Pass (p-value >= 0.01)

The sequence has appropriate approximate entropy, indicating sufficient unpredictability and randomness.

Fail (p-value < 0.01)

Approximate entropy deviates from expected. Common causes:

  • Too low entropy: Sequence is predictable from past patterns
  • Pattern repetition: Same patterns occur too frequently
  • Correlation: Future bits depend on previous bits
  • Insufficient mixing: Generator doesn't adequately randomize output
Note: This test is similar to the Serial Test but focuses on entropy measurement rather than uniformity. Both tests provide complementary information about pattern distribution.

Common Failure Patterns

Example 1: Predictable Patterns

Sequence: 01010101... Result: Very low ApEn - next bit always predictable Test: FAIL - No entropy/randomness

Example 2: Block Repetition

Repeating blocks reduce entropy ApEn too low for truly random sequence Test: FAIL - Insufficient unpredictability

When to Use This Test

This test is valuable for:

  • Entropy Assessment: Measuring information content
  • Predictability Testing: Determining if patterns reveal future bits
  • Cryptographic Validation: Ensuring high unpredictability
  • Compression Resistance: Verifying incompressibility
