Small Sequence Analysis

Uniqueness Ratio Test

Quick Summary: The Uniqueness Ratio Test measures how many distinct values appear in a sequence compared to the total count. In truly random sequences, we expect high uniqueness - excessive repetition of the same values indicates poor randomness or bias.

What It Tests For

This test detects excessive repetition in small sequences. A truly random generator should produce varied outputs, not repeatedly favor the same values. This test specifically checks:

  • Whether the generator is "stuck" producing the same values repeatedly
  • If the output range is artificially constrained
  • Whether there's a hidden bias toward specific numbers

How It Works

Step 1: Count Unique Values

The test counts how many distinct numbers appear in your sequence:

Sequence: [5, 3, 7, 3, 9, 5, 2] Total: 7 numbers Unique: 5 values (2, 3, 5, 7, 9) Uniqueness: 5/7 = 71.4%

Step 2: Compare to Threshold

The test applies different thresholds based on sequence size:

Very Small (<10 numbers): Expect ≥ 70% unique Small (10-20 numbers): Expect ≥ 65% unique Larger (20+ numbers): Expect ≥ 75% unique

These thresholds account for the natural probability of some repetition in small samples while still catching obvious problems.

Step 3: Calculate P-Value

The test computes a p-value by comparing the observed uniqueness ratio to the threshold. Higher ratios (more unique values) produce higher p-values.

Understanding the Results:
  • High Uniqueness (≥75%): EXCELLENT - Very good variety of values
  • Good Uniqueness (65-75%): PASS - Acceptable for small sequences
  • Low Uniqueness (<65%): FAIL - Too much repetition detected

Interpreting Results

Pass - High Uniqueness

Your sequence shows good variety with minimal repetition. This suggests the generator is producing a diverse range of outputs as expected from a random source.

Fail - Low Uniqueness

Too many repeated values detected. Common causes include:

  • Broken Generator: The RNG may be stuck in a short cycle
  • Seeding Issues: Poor initialization causing repeated patterns
  • Range Problems: Output constrained to too narrow a range
  • Modulo Bias: Using modulo operation causing value clustering
⚠️ Note: This test is specifically designed for small sequences (< 10,000 bits / ~300 numbers). For larger sequences, the standard NIST tests provide better statistical analysis.

Why This Test Matters

Standard NIST tests work at the bit level and need thousands of bits to detect patterns. With small sequences, NIST tests can miss obvious number-level problems like a value appearing 6 times in 18 numbers. The Uniqueness Ratio Test catches these issues that would otherwise go undetected.

Complementary Tests

This test works alongside other small-sequence tests:

  • Frequency Analysis: Checks if any single value appears too often
  • Chi-Squared: Tests overall distribution uniformity
  • Serial Correlation: Detects dependencies between consecutive values
