Small Sequence Analysis

Chi-Squared Uniformity Test

Quick Summary: The Chi-Squared Uniformity Test determines whether values are distributed evenly across the sequence. In a truly random sequence, each unique value should appear with roughly equal frequency - clustering or imbalance indicates non-randomness.

What It Tests For

This test checks for uniform distribution of values. A good random number generator should not favor certain values over others. The chi-squared statistic measures how far the observed frequencies deviate from what we'd expect if all values were equally likely.

  • Detects if certain numbers appear much more or less often than expected
  • Identifies clustering of values in specific ranges
  • Reveals generator bias toward particular outputs

How It Works

Step 1: Count Value Frequencies

The test counts how often each unique value appears:

Sequence: [5, 3, 7, 3, 9, 5, 2, 8, 5, 6] Frequencies: 2: 1 time 3: 2 times 5: 3 times ← appears most often 6: 1 time 7: 1 time 8: 1 time 9: 1 time

Step 2: Calculate Expected Frequency

If values were perfectly uniform, we'd expect:

Total numbers: 10 Unique values: 7 Expected per value: 10 / 7 = 1.43 occurrences

Step 3: Compute Chi-Squared Statistic

The test calculates how much the observed frequencies deviate from the expected:

χ² = Σ [(Observed - Expected)² / Expected] For each unique value: χ² += (actual_count - 1.43)² / 1.43

A larger chi-squared value indicates greater deviation from uniformity.

Step 4: Compare to Threshold

The test uses an adaptive threshold based on the number of unique values:

Threshold = k × 2 Where k = number of unique values For small sequences, this relaxed threshold accounts for natural variance while still catching real problems.
Understanding Results:
  • χ² < threshold: PASS - Distribution is acceptably uniform
  • χ² ≥ threshold: FAIL - Non-uniform distribution detected

The p-value indicates how close you are to the threshold: higher p-values mean better uniformity.

Interpreting Results

Pass - Uniform Distribution

Values are distributed relatively evenly. While not perfect (perfect uniformity is actually suspicious in small samples), the distribution shows no concerning bias.

Fail - Non-Uniform Distribution

Significant clustering or imbalance detected. Common causes:

  • Generator Bias: The RNG systematically favors certain output ranges
  • Modulo Bias: Using mod operator incorrectly, causing uneven distribution
  • Range Mapping Issues: Poor scaling from one range to another
  • Insufficient Mixing: Internal state not thoroughly randomized
  • Sample Size: Very small samples (< 5 numbers) skipped automatically
⚠️ Important: This test is calibrated for small sequences where standard chi-squared tables don't apply well. For larger sequences (1000+ numbers), traditional chi-squared goodness-of-fit tests provide more accurate analysis.

Example Scenarios

Example 1: Good Uniform Distribution

Numbers: [15, 42, 7, 89, 23, 56, 3, 91, 67, 12, 34, 78, 45, 98, 21, 64, 8, 73, 29, 50] Unique: 20 values (all different) Expected: 1.0 per value Observed: 1.0 per value (perfect for small sample) χ²: 0.0 Threshold: 40.0 Result: PASS ✓ - Excellent uniformity

Example 2: Poor Distribution

Numbers: [5, 5, 5, 5, 3, 3, 7, 8] Unique: 4 values Expected: 2.0 per value Observed: 3: 2 times (OK, contributes (2-2)²/2 = 0.0) 5: 4 times (2×!, contributes (4-2)²/2 = 2.0) ← problem 7: 1 time (½×, contributes (1-2)²/2 = 0.5) 8: 1 time (½×, contributes (1-2)²/2 = 0.5) χ²: 3.0 Threshold: 8.0 (k × 2 with k = 4) Result: PASS, but value 5 still shows concerning bias

Example 3: Clustered Values

Numbers: [100, 101, 102, 103, 104, 105, 106, 107, 108, 109] Unique: 10 values Distribution: Perfectly uniform... BUT all clustered in narrow range (100-109) χ²: 0.0 (uniform within the range) Result: PASS for this test, but would fail other tests Note: This shows why multiple tests are needed!

Relationship to NIST Tests

This test is conceptually similar to the NIST Frequency Test, but operates at the number level rather than bit level. While NIST checks for balanced 0s and 1s in the bit stream, this test checks for balanced distribution of actual numeric values in the sequence.

Best Practices

  • Use with at least 10-20 numbers for meaningful results
  • Combine with Frequency Analysis test to catch both overall bias and specific value repetition
  • Consider the Uniqueness Ratio test alongside this one
  • Remember that passing this test alone doesn't guarantee randomness - patterns could still exist
