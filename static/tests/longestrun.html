NIST SP 800-22 Statistical Test

Longest Run of Ones Test

Quick Summary: The Longest Run of Ones Test examines the length of the longest consecutive sequence of 1s within designated blocks of the bit stream. In a truly random sequence, the length of the longest run should follow a predictable distribution.

What It Tests For

This test detects whether the random number generator produces sequences with abnormally long or short runs of 1s. It specifically checks:

  • Whether consecutive strings of 1s are unusually long (indicating clumping or bias)
  • Whether consecutive strings of 1s are unusually short (indicating over-dispersal)
  • The consistency of run lengths across different blocks of the sequence
  • Hidden periodicities that cause repetitive patterns of runs

How It Works

Step 1: Divide Into Blocks

The bit sequence is divided into blocks of a specified length M, which depends on the total sequence length:

Sequence length >= 128: Block size M = 8 Sequence length >= 6,272: Block size M = 128 Sequence length >= 750,000: Block size M = 10,000

Step 2: Find Longest Run in Each Block

For each block, the test identifies the longest consecutive sequence of 1s:

Block 1: 11001101 11111111 00101010 11110010 Longest run of 1s: 8 consecutive ones Block 2: 10101010 11100111 00011111 10001000 Longest run of 1s: 5 consecutive ones

Step 3: Categorize Run Lengths

The longest runs are categorized into intervals. For M=8 blocks, the categories are:

  • v0: runs of length 1 or less
  • v1: runs of length 2
  • v2: runs of length 3
  • v3: runs of length 4 or more

Step 4: Compare Against Expected Distribution

The test uses a chi-square statistic to compare the observed distribution of run lengths against the theoretical distribution expected for truly random data:

Chi-Square = Sum((Observed - Expected)^2 / Expected)

The chi-square value is then converted to a p-value to determine if the deviation is statistically significant.

Understanding P-Values:
  • p >= 0.01: PASS - Run lengths appear random
  • p < 0.01: FAIL - Run lengths deviate from randomness

Interpreting Results

Pass (p-value >= 0.01)

The longest runs of 1s follow the expected distribution for random data. This indicates that your generator doesn't create unusual clustering or gaps in consecutive bits.

Fail (p-value < 0.01)

The distribution of longest runs is statistically abnormal. Common causes include:

  • Clumping Bias: The generator produces too many long runs, creating clusters of similar bits
  • Dispersal Bias: The generator produces too many short runs, avoiding consecutive similar bits
  • Block-based Generation: If the RNG generates output in fixed-size blocks, boundaries may be visible
  • Periodic Patterns: Hidden periodicities can create consistent run length patterns
Note: This test specifically examines runs of 1s, not 0s. However, in a balanced random sequence, the distribution of runs should be similar for both values. If this test passes but the Frequency Test fails, investigate bit balance issues.

Common Failure Patterns

Example 1: Excessive Long Runs

Sequence: 11111111 11110000 11111111 11100000 Analysis: Too many blocks with runs of 7-8 ones Result: FAIL - Clumping detected Cause: Generator may be "sticky" or use modulo operations

Example 2: All Short Runs

Sequence: 10101010 10101010 10101010 10101010 Analysis: No runs longer than 1 bit Result: FAIL - Over-dispersed Cause: Alternating pattern or XOR-based generation flaw

Example 3: Periodic Pattern

Sequence: 11101110 11101110 11101110 11101110 Analysis: Every block has exactly 3-bit runs Result: FAIL - Non-random periodicity Cause: Repeating pattern in generation algorithm

When to Use This Test

Use this test when:

  • Validating generators used for cryptographic purposes where bit patterns must be unpredictable
  • Checking for clumping issues that could reveal generator state
  • Testing hardware RNGs that might have correlation issues between consecutive outputs
  • Verifying that post-processing (like whitening) hasn't introduced artifacts

This test is particularly effective at detecting linear congruential generators (LCGs) and other generators with correlation between consecutive outputs.
