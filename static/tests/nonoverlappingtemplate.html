NIST SP 800-22 Statistical Test

Non-overlapping Template Matching Test

Quick Summary: The Non-overlapping Template Matching Test searches for specific m-bit patterns (templates) within the bit sequence. It counts how often these patterns appear in non-overlapping windows and checks whether the frequency matches what would be expected from truly random data.

What It Tests For

This test detects whether specific bit patterns occur too frequently or too rarely in the sequence. It checks for:

  • Pattern bias: Generator tendency to produce (or avoid) certain bit combinations
  • Template imbalance: Uneven distribution of specific m-bit patterns
  • Generator artifacts: Patterns that appear due to algorithm structure
  • Non-periodic repetitions: Patterns that repeat but not at fixed intervals

How It Works

Step 1: Select Templates

The test uses predefined m-bit templates (typically m=9 or m=10). Common templates include:

For m=9: Template: 000000001 (8 zeros followed by 1) Template: 011111111 (0 followed by 8 ones) ... (148 different 9-bit non-periodic templates)

Step 2: Divide Sequence Into Blocks

The sequence is divided into N non-overlapping blocks of length M bits each:

Sequence: [Block 1: M bits][Block 2: M bits]...[Block N: M bits] Typical block size M = 968 bits for m=9 templates Number of blocks N = 8 (minimum)

Step 3: Count Template Matches (Non-overlapping)

Within each block, count how many times the template appears, using non-overlapping matching:

Block: 10100110111011101... Template: 111 (looking for "111") Search process: 10[1]00110[111]011101... ^ ^ match found, skip ahead Skip overlapping positions after each match

Step 4: Calculate Chi-Square Statistic

For each template, compute a chi-square statistic comparing observed vs expected matches:

μ = (M - m + 1) / 2^m (expected matches per block) σ² = M * ((1/2^m) - (2m-1)/2^(2m)) (variance) Chi-Square = Sum((Wi - μ)² / σ²) for all N blocks Where Wi = number of template matches in block i

Step 5: Compute P-Value

The chi-square statistic is converted to a p-value using the incomplete gamma function.

Understanding P-Values:
  • p >= 0.01: PASS - Template frequency appears random
  • p < 0.01: FAIL - Template distribution is abnormal

The test produces multiple p-values (one per template tested).

Interpreting Results

Pass (p-value >= 0.01)

The template appears with expected frequency across all blocks. This indicates:

  • No bias toward or against this specific pattern
  • Pattern distribution is consistent with randomness
  • Generator doesn't have structural weaknesses related to this template

Fail (p-value < 0.01)

The template frequency deviates significantly from random expectations. Common causes include:

  • Too Many Matches: Generator favors producing this pattern
  • Too Few Matches: Generator systematically avoids this pattern
  • Uneven Distribution: Pattern clusters in some blocks but not others
  • Algorithm Artifacts: Generator structure creates/prevents certain bit combinations
Important: This test requires sufficient sequence length. For m=9 templates with N=8 blocks and M=968 bits per block, you need at least 7,744 bits. Shorter sequences will not be tested reliably.

Common Failure Patterns

Example 1: Excessive Pattern Matches

Template: 11111111 (8 ones) Expected: ~3.8 matches per block Observed: 15-20 matches per block Result: FAIL - Too many matches Cause: Generator produces long runs of 1s

Example 2: Pattern Avoidance

Template: 10101010 (alternating) Expected: ~3.8 matches per block Observed: 0-1 matches per block Result: FAIL - Pattern avoided Cause: Generator doesn't produce alternating patterns

Example 3: Clustered Distribution

Template: 00001111 Expected: Uniform distribution across blocks Observed: Many matches in blocks 1-3, none in blocks 4-8 Result: FAIL - Non-uniform distribution Cause: Generator behavior changes over time

Non-overlapping vs Overlapping

This test uses non-overlapping matching, meaning after finding a match, the search continues after the matched pattern:

Sequence: 111111111 Template: 111 Non-overlapping: Finds 3 matches (positions 0, 3, 6) [111][111][111] Overlapping would find 7 matches (positions 0-6) [111]11111 1[111]1111 11[111]111 ... etc

The non-overlapping approach better reflects independent occurrences of patterns.

When to Use This Test

This test is valuable for:

  • Cryptographic RNGs: Ensuring no exploitable pattern biases
  • Hardware RNGs: Detecting biases in physical entropy sources
  • Algorithm Validation: Finding structural weaknesses in PRNG algorithms
  • Post-Processing Verification: Ensuring whitening didn't introduce new biases

Because the test examines 148 different templates, it provides broad coverage of potential pattern biases.
