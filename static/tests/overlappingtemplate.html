NIST SP 800-22 Statistical Test

Overlapping Template Matching Test

Quick Summary: The Overlapping Template Matching Test searches for a specific m-bit pattern (typically all 1s) and counts occurrences using overlapping windows. Unlike the non-overlapping version, this test allows consecutive matches at adjacent positions, detecting clustering of specific patterns.

What It Tests For

This test specifically detects clustering behavior and run properties by counting overlapping occurrences of a target template (usually m consecutive 1s). It checks for:

  • Pattern clustering: Whether the pattern appears in clumps rather than randomly distributed
  • Run length deviations: Abnormally long or short runs of 1s
  • Local correlations: Tendency for 1s to cluster together
  • Generator memory: Whether previous outputs influence subsequent outputs

How It Works

Step 1: Define the Template

The standard template is m consecutive 1s (typically m=9 or m=10):

For m=9: Template: 111111111 (nine consecutive 1s)

Step 2: Divide Into Blocks

The sequence is divided into N blocks of length M bits each:

Typical configuration: Block length M = 1,032 bits Number of blocks N = 968 (minimum sequence: ~1 million bits)

Step 3: Count Overlapping Matches

Within each block, count template matches using a sliding window that moves one position at a time:

Block: 111111111011... Template: 11111 (m=5 for illustration) Position 0: [11111]1111011 - MATCH Position 1: 1[11111]111011 - MATCH Position 2: 11[11111]11011 - MATCH Position 3: 111[11111]1011 - MATCH Position 4: 1111[11111]011 - MATCH Position 5: 11111[1]1110... - NO MATCH Total: 5 overlapping matches

Step 4: Categorize Match Counts

Count how many blocks have 0, 1, 2, 3, 4, or 5+ matches:

v0 = number of blocks with 0 matches v1 = number of blocks with 1 match v2 = number of blocks with 2 matches v3 = number of blocks with 3 matches v4 = number of blocks with 4 matches v5 = number of blocks with 5+ matches

Step 5: Chi-Square Test

Compare the observed distribution against theoretical probabilities:

Chi-Square = Sum((vi - N*πi)² / (N*πi)) for i=0 to 5 Where πi = theoretical probability for i matches
Understanding P-Values:
  • p >= 0.01: PASS - Match distribution appears random
  • p < 0.01: FAIL - Pattern clustering is abnormal

Interpreting Results

Pass (p-value >= 0.01)

The distribution of overlapping template matches is consistent with randomness. This indicates:

  • Runs of 1s occur at expected frequencies
  • No unusual clustering of the pattern
  • Generator doesn't have memory effects for this pattern

Fail (p-value < 0.01)

The match distribution is statistically abnormal. Common causes include:

  • Excessive Clustering: Too many blocks with multiple matches (pattern clumping)
  • Pattern Avoidance: Too many blocks with zero matches
  • Generator Stickiness: Once producing 1s, continues producing them
  • Run Length Bias: Runs of 1s are consistently too long or too short
Note: This test is specifically designed for the "all 1s" template and examines overlapping occurrences. It's complementary to the Longest Run test and provides additional information about clustering behavior.

Common Failure Patterns

Example 1: Excessive Clustering

Observation: Many blocks have 5+ overlapping matches Analysis: Long runs of 1s are too common Result: FAIL - Pattern clusters excessively Cause: Generator "sticks" on high values

Example 2: Pattern Avoidance

Observation: Nearly all blocks have 0 matches Analysis: Template rarely or never appears Result: FAIL - Generator avoids pattern Cause: Insufficient randomness or bias against 1s

Example 3: Uniform Pattern Distribution

Observation: Every block has exactly 1-2 matches Analysis: Too uniform, lacks natural variation Result: FAIL - Artificially regular Cause: Post-processing that enforces uniformity

Overlapping vs Non-overlapping

The key difference from the non-overlapping test:

Sequence: 1111111111 (ten 1s) Template: 111111 (six 1s) Overlapping count: 5 matches [111111]1111 1[111111]111 11[111111]11 111[111111]1 1111[111111] Non-overlapping count: 1 match [111111]1111 (then skip to position 7)

Overlapping matching is more sensitive to clustering and run length issues.

When to Use This Test

This test is particularly valuable for:

  • Detecting Run Clustering: Finding generators that produce clumped runs
  • Memory Effects: Identifying correlation between consecutive outputs
  • Hardware RNGs: Detecting physical phenomena that cause bit clustering
  • Post-Processing Validation: Ensuring whitening preserves local randomness

Use this test alongside the Longest Run test for comprehensive run analysis.

Mathematical Background

The theoretical probabilities πi for match counts follow from the analysis of overlapping pattern occurrences in random binary sequences. For a random sequence, the distribution of overlapping matches in blocks follows a known probability distribution that depends on the template length m and block size M.
