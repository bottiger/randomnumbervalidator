NIST SP 800-22 Statistical Test

Maurer's Universal Statistical Test

Quick Summary: Maurer's Universal Statistical Test measures the compressibility of the sequence by tracking the distance between repeating L-bit patterns. Truly random sequences are incompressible and should show predictable spacing between pattern repetitions.

What It Tests For

This test detects whether the sequence can be significantly compressed, which would indicate non-randomness. It specifically checks for:

  • Compressibility: Whether the data contains patterns that allow compression
  • Pattern repetition: How quickly patterns repeat in the sequence
  • Entropy deficiency: Insufficient randomness that creates predictable structure
  • Generator state space: Whether the RNG has adequate state diversity

How It Works

Step 1: Choose Block Size L

The sequence is divided into L-bit blocks. The block size depends on the sequence length:

Sequence length >= 387,840 bits: L = 6 Sequence length >= 904,960 bits: L = 7 Sequence length >= 2,068,480 bits: L = 8 ...up to L = 16 for very long sequences

Step 2: Initialize Dictionary (Learning Phase)

The first Q = 10×2^L blocks form an initialization segment where we build a "dictionary" tracking the last occurrence of each possible L-bit pattern:

For L=6 (64 possible patterns): Q = 10 × 64 = 640 blocks used for initialization Pattern 000000 last seen at position: 234 Pattern 000001 last seen at position: 567 ... Pattern 111111 last seen at position: 602

Step 3: Test Phase - Measure Distances

For the remaining K blocks, measure the distance from each block to the last occurrence of that same pattern:

Position 650: Block = 010101 Last occurrence of 010101 was at position 590 Distance = 650 - 590 = 60 blocks Log2(distance) = log2(60) ≈ 5.9

Step 4: Calculate Test Statistic

Compute the average log-distance across all K test blocks:

f = (1/K) × Sum(log2(distance[i])) for i=1 to K Where distance[i] = blocks since last occurrence of pattern i

Step 5: Compare to Expected Value

For truly random data, the expected value and variance are known theoretically. The test statistic is:

P-value based on: Observed: f Expected: expectedValue(L) Variance: expectedVariance(L, K)
Understanding P-Values:
  • p >= 0.01: PASS - Sequence appears incompressible (random)
  • p < 0.01: FAIL - Sequence is too compressible (non-random)

Interpreting Results

Pass (p-value >= 0.01)

The sequence shows expected pattern repetition distances, indicating:

  • Sequence is effectively incompressible
  • Patterns repeat at random-like intervals
  • Generator has sufficient entropy
  • No obvious structural patterns

Fail (p-value < 0.01)

The sequence is more compressible than random data should be. Common causes include:

  • Too Compressible (f too low): Patterns repeat too frequently
  • Too Incompressible (f too high): Patterns never or rarely repeat
  • Short Generator Period: Limited state space causes early pattern repetition
  • Bias in Patterns: Some patterns occur much more frequently than others
  • Correlation: Recent outputs influence future outputs predictably
Important: This test requires substantial data - at minimum 387,840 bits (~48KB) for L=6. Longer sequences allow testing with larger L values, which can detect more subtle non-randomness.

Common Failure Patterns

Example 1: Frequent Pattern Repetition

Observation: f = 3.2 (expected ~5.2 for L=6) Analysis: Patterns repeat much more frequently than expected Result: FAIL - Too compressible Cause: Generator has limited state or strong correlations

Example 2: No Pattern Repetition

Observation: f = 8.5 (expected ~5.2 for L=6) Analysis: Patterns almost never repeat Result: FAIL - Artificially diverse Cause: Generator avoids repetitions (counter-based)

Example 3: Short PRNG Cycle

Generator: LCG with period 2^16 Observation: Patterns start repeating at exactly 2^16 intervals Result: FAIL - Periodic structure detected Cause: Generator period is too short

Connection to Compression

This test is fundamentally based on data compression theory:

Compressible data = Non-random data Incompressible data = Random data The test simulates dictionary-based compression by: 1. Building a table of pattern positions 2. Measuring how far you must "look back" to find repeats 3. Averaging these distances Short average distances → Data compresses well → FAIL Expected distances → Data incompressible → PASS

When to Use This Test

This test is particularly valuable for:

  • Large Datasets: Requires substantial data but provides powerful assessment
  • Entropy Validation: Directly measures information-theoretic randomness
  • Cryptographic RNGs: Ensures output can't be compressed/predicted
  • Generator Period Testing: Detects insufficient state space
  • Hardware RNG Validation: Identifies entropy source deficiencies

Mathematical Background

Maurer's test is based on information theory and the relationship between randomness and Kolmogorov complexity. A truly random sequence is incompressible - you cannot represent it more compactly without loss. The test measures compressibility indirectly by examining pattern repetition intervals, which relate to the entropy of the source.
