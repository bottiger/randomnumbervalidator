NIST SP 800-22 Statistical Test

Linear Complexity Test

Quick Summary: The Linear Complexity Test determines the length of the shortest Linear Feedback Shift Register (LFSR) that could generate a given sequence. For truly random sequences, the linear complexity should be close to half the sequence length.

What It Tests For

This test detects whether the sequence could be generated by a Linear Feedback Shift Register (LFSR), which would make it cryptographically weak. It checks for:

  • Linear structure: Patterns that follow linear recurrence relations
  • LFSR-based generation: Whether an LFSR could produce the sequence
  • Predictability: Sequences that can be predicted with linear algebra
  • Insufficient complexity: Sequences simpler than truly random data

How It Works

Step 1: Divide Into Blocks

The sequence is divided into M non-overlapping blocks of length N bits each (typically N = 500 or N = 1000):

Block size N = 500 bits Minimum M = 200 blocks Total bits needed: 100,000 minimum

Step 2: Calculate Linear Complexity for Each Block

For each N-bit block, calculate the linear complexity using the Berlekamp-Massey algorithm. Linear complexity L is the length of the shortest LFSR that can generate the sequence:

Sequence: 1101001... Linear Complexity L = length of shortest LFSR L = 0: Only all-zeros sequence L = 1: Simple repeating pattern (e.g., 000... or 111...) L = N/2: Expected for random sequence L = N: Maximum complexity (ideal random)

Step 3: Calculate Deviation

For each block, compute how much the linear complexity deviates from the expected value:

μ = N/2 + (9 + (-1)^(N+1)) / 36 (expected linear complexity) T = (-1)^N × (L - μ) + 2/9 Where L = observed linear complexity for the block

Step 4: Categorize Deviations

Count how many blocks fall into each deviation category:

v0 = blocks with T ≤ -2.5 v1 = blocks with -2.5 < T ≤ -1.5 v2 = blocks with -1.5 < T ≤ -0.5 v3 = blocks with -0.5 < T ≤ 0.5 v4 = blocks with 0.5 < T ≤ 1.5 v5 = blocks with 1.5 < T ≤ 2.5 v6 = blocks with T > 2.5

Step 5: Chi-Square Test

Compare the observed distribution against expected probabilities:

Chi-Square = Sum((vi - M×πi)² / (M×πi)) for i=0 to 6 Where πi are theoretical probabilities for each category
Understanding P-Values:
  • p >= 0.01: PASS - Linear complexity distribution appears random
  • p < 0.01: FAIL - Complexity deviates from randomness

Interpreting Results

Pass (p-value >= 0.01)

The distribution of linear complexities matches random expectations. This indicates:

  • Sequence is not generated by a simple LFSR
  • No obvious linear structure
  • Sequence resists linear prediction methods
  • Cryptographically acceptable complexity

Fail (p-value < 0.01)

Linear complexity distribution is abnormal. Common causes include:

  • Too Low Complexity: Sequence can be generated by short LFSR (predictable)
  • Too High Complexity: Artificially complex (possibly post-processed poorly)
  • LFSR Generator: Directly using an LFSR to generate random numbers
  • Linear Congruential Generator: Some LCGs exhibit linear properties
  • Stream Cipher Weakness: Linear components in cipher design
Important: Low linear complexity is a serious security flaw. If this test fails with complexity too low, the generator should NOT be used for cryptographic purposes as it's vulnerable to linear prediction attacks.

Common Failure Patterns

Example 1: LFSR-Based Generator

Generator: 16-bit LFSR Observation: Nearly all blocks have L ≤ 20 Expected: L ≈ 250 for N=500 Result: FAIL - Linear complexity far too low Cause: Generator IS an LFSR

Example 2: Simple Pattern

Sequence: 01010101010101... Observation: L = 2 for all blocks Expected: L ≈ 250 for N=500 Result: FAIL - Trivially simple pattern Cause: Alternating or repeating structure

Example 3: Artificially High Complexity

Observation: Most blocks have L very close to N Analysis: Unnaturally high complexity Result: FAIL - Distribution too uniform Cause: Poor post-processing or non-random "randomization"

What is Linear Complexity?

Linear complexity is the length of the shortest LFSR needed to generate a sequence:

Example sequence: 1,1,0,1,1,0,1,1,0... Pattern: Repeats "110" Linear Complexity: 3 (can be generated by 3-bit LFSR) Random sequence: 1,0,1,1,0,0,1,0,1,1,1... No simple pattern Linear Complexity: ≈ N/2 (for N-bit sequence)

The Berlekamp-Massey algorithm efficiently computes this value.

When to Use This Test

This test is critical for:

  • Cryptographic Applications: Essential for ensuring resistance to linear attacks
  • Stream Cipher Validation: Verifying cipher output isn't linearly predictable
  • LFSR Detection: Identifying generators based on linear feedback
  • Security-Critical RNGs: Any generator used for cryptographic keys or nonces
  • Hardware RNG Validation: Ensuring physical sources don't have linear biases

Failure of this test is a critical security vulnerability.

Mathematical Background

The linear complexity of a sequence is related to its predictability via linear methods. The Berlekamp-Massey algorithm finds the shortest LFSR that generates a given sequence, which directly relates to how many bits an attacker needs to observe before predicting future outputs. For cryptographic security, high linear complexity is essential.
