NIST SP 800-22 Statistical Test

Random Excursion Variant Test

Quick Summary: The Random Excursion Variant Test is similar to the Random Excursions Test but uses a simpler counting method. It tracks the total number of times the random walk visits each state across the entire sequence, without requiring complete cycles.

What It Tests For

This variant test detects deviations in how often the cumulative sum random walk visits different states:

  • State visit frequency: Whether certain states are visited too often or rarely
  • Random walk balance: Symmetry in positive vs negative state visits
  • Cumulative sum behavior: Overall properties of the random walk
  • Simpler cycle analysis: Less stringent than full Random Excursions Test

How It Works

Step 1: Create Random Walk

Convert bits to +1/-1 and compute cumulative sum:

Binary: 1 0 1 1 0 0 1 0 Mapped: +1 -1 +1 +1 -1 -1 +1 -1 Cusum: +1 0 +1 +2 +1 0 +1 0

Step 2: Count Total State Visits

For 18 states (x = -9 to +9, excluding 0), count total visits across entire sequence:

State +1: Count how many times cusum equals +1 State +2: Count how many times cusum equals +2 State -1: Count how many times cusum equals -1 ... etc for all 18 states

Step 3: Calculate Test Statistic

For each state x, compare observed visits to expected:

J = number of times random walk returns to 0 ξ(x) = total visits to state x Test statistic = (ξ(x) - J) / sqrt(2J × (4|x| - 2))

Step 4: Compute P-Values

Convert test statistic to p-value for each of the 18 states.

Understanding P-Values:
  • p >= 0.01: PASS - State visits appear random
  • p < 0.01: FAIL - Abnormal visit frequency

The test produces 18 p-values (one per state), ideally most should pass.

Interpreting Results

Pass (p-values >= 0.01)

State visit frequencies match random expectations, indicating proper random walk distribution.

Fail (p-values < 0.01)

Abnormal state visitation pattern. Common causes:

  • State bias: Random walk favors or avoids certain states
  • Asymmetric walk: Positive and negative states have different visit frequencies
  • Insufficient returns to zero: Walk doesn't return to 0 often enough
  • Correlation effects: Bit dependencies create biased random walk
Note: This variant is less stringent than the Random Excursions Test and doesn't require complete cycles. It's useful when sequences have few complete excursions but still provides valuable random walk analysis.

Common Failure Patterns

Example 1: Positive State Bias

Positive states (+1, +2, +3) visited too frequently Negative states (-1, -2, -3) visited too rarely Result: Fails for multiple states Cause: Bias toward 1s in the sequence

Example 2: Near-Zero Clustering

States +1 and -1 visited excessively Distant states rarely visited Result: Fails for states near zero Cause: Cumulative sum oscillates around zero too much

Example 3: State Avoidance

Certain states (e.g., +5, -5) almost never visited Result: Fails for those specific states Cause: Generator has structure preventing certain cumulative sums

Variant vs Standard Random Excursions

Key differences from the standard Random Excursions Test:

Standard Random Excursions: - Analyzes complete cycles (0 → states → 0) - Requires sufficient complete cycles (~500+) - Tests 8 states - More stringent Random Excursion Variant: - Counts total visits to states (no cycle requirement) - Works with fewer cycles - Tests 18 states (-9 to +9) - More lenient, complementary information

When to Use This Test

This test is valuable for:

  • Complementing Standard Test: Use with Random Excursions Test
  • Fewer Complete Cycles: When standard test can't run
  • Broader State Coverage: Tests more states than standard version
  • Random Walk Validation: Verifying overall walk properties
