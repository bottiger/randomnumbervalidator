NIST SP 800-22 Statistical Test

Cumulative Sums (Cusums) Test

Quick Summary: The Cumulative Sums Test computes the running cumulative sum of the sequence (treating 0 as -1 and 1 as +1) and checks whether this random walk deviates too far from zero. Large deviations indicate bias toward zeros or ones.

What It Tests For

This test detects bias in the sequence by tracking cumulative deviations. It's particularly sensitive to:

  • Early/late bias: More 0s or 1s at beginning or end of sequence
  • Drift: Gradual shift in bit balance over time
  • Position-dependent bias: Bias that varies by location in sequence
  • Local imbalance: Regions with too many 0s or 1s

How It Works

Step 1: Convert to +1/-1

Transform the bit sequence so 0 becomes -1 and 1 stays +1:

Binary: 1 0 1 1 0 0 1 0 Mapped: +1 -1 +1 +1 -1 -1 +1 -1

Step 2: Compute Cumulative Sums

Calculate running sum in both forward and reverse directions:

Forward cusum: S[k] = Sum of bits 1 to k Reverse cusum: S'[k] = Sum of bits k to n Example (forward): Position: 1 2 3 4 5 6 7 8 Value: +1 -1 +1 +1 -1 -1 +1 -1 Cusum: +1 0 +1 +2 +1 0 +1 0

Step 3: Find Maximum Deviation

Find the largest absolute cumulative sum value:

z_forward = max(|S[k]|) for all k z_reverse = max(|S'[k]|) for all k

Step 4: Calculate P-Values

Convert both forward and reverse statistics to p-values. Large deviations produce small p-values.

Understanding P-Values:
  • p >= 0.01: PASS - Cumulative sums within expected range
  • p < 0.01: FAIL - Excessive deviation detected

The test produces two p-values (forward and reverse), both should pass.

Interpreting Results

Pass (p-values >= 0.01)

Cumulative sums stay within expected bounds, indicating good bit balance throughout the sequence.

Fail (p-values < 0.01)

Excessive cumulative deviation detected. Common causes:

  • Early bias: Sequence starts with too many 0s or 1s
  • Late bias: Sequence ends with imbalanced bits
  • Generator drift: Bias increases over generator runtime
  • Segmented generation: Different segments have different biases
Note: This test is similar to the Frequency Test but more sensitive to position-dependent bias. A sequence could pass the Frequency Test (overall balanced) but fail Cusums (localized imbalance).

Common Failure Patterns

Example 1: Front-Loaded Ones

Sequence: 11111111000000... Forward cusum reaches +8 then drops Test: FAIL (forward) - Early bias toward 1s

Example 2: Gradual Drift

Generator slowly shifts from producing 45% ones to 55% ones Cumulative sum drifts positive Test: FAIL - Drift detected

Example 3: Back-Loaded Zeros

Sequence: ...00000000 Reverse cusum shows large negative value Test: FAIL (reverse) - Late bias toward 0s

Technical Details

Test Type
Random Walk
Directions
Forward & Reverse
Min Bits
100
Significance
α = 0.01

When to Use This Test

This test is valuable for:

  • Position-Dependent Bias: Finding imbalance that varies by location
  • Generator Warm-up Issues: Detecting initialization bias
  • Generator Degradation: Finding drift over time
  • Complementing Frequency Test: More sensitive to local imbalance
