About This Generator

            

                This implements a simplified Mersenne Twister (MT19937) - the same algorithm used
                in old PHP mt_rand(). However, this version uses incomplete tempering
                (only 2 of 4 steps), which introduces subtle statistical weaknesses.
            

            

                The numbers look completely random to human inspection and pass basic distribution checks, but
                fail advanced statistical tests. This is similar to how many real-world implementations have
                small bugs or simplifications that compromise quality.
            


            
How It Works

            

                Mersenne Twister generates numbers through a twist operation on a 624-element state array,
                followed by tempering (bit mixing) to improve output quality. The standard implementation uses:

                By skipping the last two tempering steps, we introduce correlation patterns that NIST tests can detect.
            


            
Known Weaknesses

            
    
                
  • Incomplete tempering: Missing bit-mixing steps leave correlations in output
    • 
                
  • Modulo bias: Simple range reduction introduces distribution bias
    • 
                
  • State recovery: With enough output, the internal state can be reconstructed
    • 
                
  • Linear artifacts: Fails linear complexity and matrix rank tests
    • 
                
  • Not cryptographically secure: Fully deterministic and reversible
    • 
                
  • Poor seeding: Simple seed initialization limits initial randomness
