About This Generator

            

                This tool provides two types of random number generators for testing purposes:
            


            
Good RNG (Cryptographically Secure)

            

                Uses the browser's crypto.getRandomValues() which provides cryptographically
                secure random numbers. This implementation:
            

            
    
                
  • High entropy: Uses the operating system's secure random source
    • 
                
  • Unpredictable: Cannot be reproduced or predicted
    • 
                
  • Suitable for security: Safe for cryptographic applications
    • 
                
  • Should pass all tests: Designed to pass rigorous statistical tests
    • 
            


            
Poor RNG (Simplified Mersenne Twister)

            

                Implements a simplified Mersenne Twister (MT19937) with incomplete tempering
                (only 2 of 4 steps). The numbers look random to human inspection but have subtle statistical weaknesses.
            


            
Known Weaknesses:

            
    
                
  • Incomplete tempering: Missing bit-mixing steps leave correlations in output
    • 
                
  • Modulo bias: Simple range reduction introduces distribution bias
    • 
                
  • Linear artifacts: Fails linear complexity and matrix rank tests
    • 
                
  • Not cryptographically secure: Fully deterministic and reversible
    • 
            


            
Why Use Both?

            

                Compare results between good and poor RNG to understand how statistical tests detect quality issues.
                The poor RNG represents a realistic "almost good" implementation - the kind you might encounter from
                a buggy implementation or simplified version.
